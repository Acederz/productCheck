# 市场数据分类管理平台 — 系统设计

> 依据《需求说明.md》与《需求确认问题.md》已确认答案整理。  
> 技术栈：**Flask + MySQL 8.0 + Vue 3**，前后端分离。

---

## 1. 系统目标

帮助管理员导入待分类商品数据，分配给操作人员填写分类字段，审核通过后写入正式库，全程留痕、可导出。

**第一期范围**：登录、用户管理、导入、查询、分配、分类填写、提交审核、正式数据、导出、日志。  
**不含**：统计看板（后续迭代）。

## 3. 总体架构

```
┌─────────────┐     HTTPS/HTTP      ┌──────────────────┐
│  浏览器      │ ◄─────────────────► │  Nginx（可选）    │
│  Vue 3 SPA  │                     └────────┬─────────┘
└─────────────┘                              │
                                    ┌────────▼─────────┐
                                    │  Flask API       │
                                    │  - 认证/权限      │
                                    │  - 业务逻辑       │
                                    │  - 文件导入导出   │
                                    └────────┬─────────┘
                                             │
                          ┌──────────────────┼──────────────────┐
                          │                  │                  │
                   ┌──────▼──────┐   ┌───────▼───────┐  ┌──────▼──────┐
                   │  MySQL 8.0  │   │  文件目录      │  │  Redis      │
                   │  业务+日志   │   │  上传Excel等   │  │  （可选）   │
                   └─────────────┘   └───────────────┘  └─────────────┘
```

**说明**：
- 前端 Vue 3 + Element Plus（或类似组件库），适合大表格、级联下拉、批量操作。
- Flask 提供 RESTful JSON API；Session 或 JWT 做登录态（内网建议 Session + Redis，简单可用 JWT）。
- 大文件导入（最高约 10 万行）建议**异步任务**：上传后立即返回批次号，后台 Celery/线程池解析入库。
- 原始 Excel、错误报告存磁盘，**180 天**后定时清理文件；数据库记录保留。

## 4. 数据状态机

```
                    ┌──────────┐
         导入成功 ──►│  未分发   │
                    └────┬─────┘
                         │ 管理员分配
                    ┌────▼─────┐
              ┌────►│  待处理   │◄────┐
              │     └────┬─────┘     │ 驳回后重新编辑
              │          │ 操作员提交 │
              │     ┌────▼─────┐     │
              │     │  待审核   │     │
              │     └────┬─────┘     │
              │    通过  │  驳回     │
              │     ┌────▼─────┐     │
              │     │  已通过   │     └──► 已驳回
              │     └──────────┘
              │ 重新分配（待处理/已驳回）
              └─ 待审核需先「撤回」→ 待处理
```

| 状态 | 操作员能否编辑 | 说明 |
|------|----------------|------|
| 未分发 | ❌ | 仅管理员可见、可分配 |
| 待处理 | ✅ | 可暂存、可提交 |
| 待审核 | ❌ | 禁止修改；需管理员撤回 |
| 已驳回 | ✅ | 修改后重新提交；规则取最新 |
| 已通过 | ❌ | 数据在正式表；任务表只读 |

**分配规则**：
- 可批量分配；`待处理`、`已驳回` 可改派操作员。
- `待审核` 须管理员先**撤回**（→ `待处理`）再改派。
- 每次分配写入分配记录表。

## 6. 分类填写业务规则

### 6.1 级联与控件

路径：`大类 → 区隔(多选) → 类别 → 主材质 → 辅材质 → 包装方式 → 尺寸 → 卷数 → 总入数`

| 字段 | 控件 | 选项来源 |
|------|------|----------|
| 是否经营 | 下拉 | 固定：是 / 否 |
| 大类 | 下拉 | Sheet「分类规则」去重 |
| 区隔 | **多选**下拉 | 同上，按已选大类过滤 |
| 类别 | 下拉 | 按 大类 + 区隔（任一匹配）过滤 |
| 主材质 | 下拉 | 按上级路径过滤 |
| 辅材质 | 下拉或文本 | 默认下拉；Sheet「辅材质」中 mode=0 的路径改文本框 |
| 包装方式 | 下拉或文本 | 默认下拉；Sheet「包装方式」中 mode=0 的路径改文本框 |
| 尺寸 | 下拉或文本 | 见 §6.4 |
| 卷数 | 下拉或文本 | 见 §6.4 |
| 总入数 | 下拉或文本 | 见 §6.4 |

文本框 placeholder 取对应 Sheet 最后一列「备注」内容。

### 6.2 「是否经营」

- **否**：分类字段可不填，允许直接提交。
- **是**：须按规则填完必填分类字段（至少到当前路径要求的层级）。

### 6.3 前端级联 API

**获取固定层级选项**（大类～包装方式）：
```
GET /api/rules/options?field=类别&path={"大类":"保鲜袋类","区隔":["保鲜袋"]}
→ { "options": ["平口", "..."], "input_mode": "select" }
```

**获取尺寸/卷数/总入数的控件类型与选项**：
```
GET /api/rules/field-meta?field=卷数&path={...已选大类～尺寸...}
→ { "input_mode": "text|select", "options": [...], "hint": "包数、盒数或卷数" }
```

### 6.4 尺寸 / 卷数 / 总入数：下拉 vs 文本框（核心逻辑）

Excel 中有两套信息，系统按以下优先级合并：

**第一步：读主表「分类规则」当前路径**

用户选完 `大类～包装方式` 后，在主表中定位匹配行集合：

| 主表列 | 含义 | 样例 |
|--------|------|------|
| 尺寸 | **两种情况**（见下） | 具体值 `XXL`；或 `0`/`1` 表示走 Sheet「尺寸」 |
| 卷数 | **输入模式**（非填写值） | `0`=文本框，`1`=下拉 |
| 总入数 | **输入模式** | `0`=文本框，`1`=下拉 |

- **尺寸列 — 情况 A（有具体选项值）**：同一路径（大类～包装方式）下「尺寸」列为 `XXL`、`6-10cm` 等，去重后作为下拉列表。
- **尺寸列 — 情况 B（值为 `0`/`1`）**（**已与业务确认**）：表示该路径**没有**内置尺寸选项，尺寸控件类型改由 Sheet「尺寸」按路径匹配决定（`1`=下拉，`0`=文本框 + hint）。
- **卷数/总入数模式**：同一路径下取唯一 mode 值（主表样例中多为 `卷数=0, 总入数=0` 表示手工填写）。

**第二步：读补充 Sheet（当 mode=1 或需 hint 时）**

| Sheet | 行数 | A 列含义 | 作用 |
|-------|------|----------|------|
| 尺寸 | 243 | 部分行 A=1/0 | mode=1 时提供尺寸下拉；mode=0 时提供文本 hint |
| 卷数 | 21 | `格式（1为下拉选择，0为填写）` | 按「大类+区隔…」前缀匹配 |
| 总入数 | 265 | 同上 | 同上 |
| 辅材质 | 1 | 同上 | 如「垃圾袋类」下辅材质为文本，hint：厚度说明 |
| 包装方式 | 3 | 同上 | 如「保鲜膜类」下包装方式为文本，hint：宽幅说明 |

匹配规则：用已选路径从「大类」起逐级匹配 Sheet 中的列；**至少匹配到大类+区隔**；更长的前缀优先。

**第三步：渲染控件**

```
若 field 为 尺寸:
  若主表该路径下存在具体尺寸选项值（非 0/1）→ 下拉（选项来自主表）
  若主表该路径下尺寸列为 0/1 → 按 Sheet「尺寸」匹配路径：
      mode=1 → 下拉（选项来自 Sheet）
      mode=0 → 文本框 + hint

若 field 为 卷数/总入数:
  若主表对应列 mode=1 → 下拉（选项来自对应 Sheet，按路径过滤）
  若 mode=0 → 文本框 + hint（优先取对应 Sheet 的备注，无则空）
```

**样例理解**（保鲜袋类）：
- 路径 `保鲜袋类/保鲜袋/平口/HDPE/点断/盒装` → 尺寸下拉 `[XXL, XL, L, M, S, XS, XXS, 一对多]`，卷数/总入数均为**文本框**（主表标记 0）。

**样例理解**（烹饪用品类）：
- 路径 `烹饪用品类/空气炸锅托盘` → 尺寸 Sheet mode=1，下拉 `≤5cm, 6-10cm, …`；总入数 Sheet mode=1，下拉 `≤5, 6~10, …`。

```
    │   └── 分类表单（级联、多选、暂存提示）
    └── （可选）个人改密
```

**列表展示字段**：与 Excel 第一行表头一致（19 列），另加状态、操作员等管理列。

**图片展示**：URL 以 `//` 开头补 `https:`；缩略图 + 点击大图；失败显示占位图。  
文描图 JSON 数组同理，支持多图预览。

**大表性能**：虚拟滚动、分页（建议每页 50~100）、列固定（ID、名称、状态）。

## 9. 暂存机制（防丢失）

```
用户编辑字段
    │
    ├─► 防抖 500ms → localStorage（key: userId_taskId）
    │
    └─► 切换行 / 失焦 / 定时 30s → POST /api/tasks/{id}/draft
              │
页面加载 ─────► 优先读服务端 draft，无则读 localStorage，再无则读正式字段
提交成功 ─────► 清除 draft + localStorage
```
