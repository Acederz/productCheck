# 常用命令速查（复制粘贴用）

按场景分类。每条命令上面有说明，整段复制到终端即可。

默认路径：
- Windows 项目：`c:\Users\acederz\Documents\worksoft\pythonProject\productCheck`
- Linux 项目：`/home/topuser/productCheck`

---

## 一、Windows：提交并推送到 GitHub

```bat
REM 进入项目目录
cd /d c:\Users\acederz\Documents\worksoft\pythonProject\productCheck

REM 查看改了哪些文件
git status

REM 把所有改动加入待提交
git add .

REM 提交（引号里改成你的说明）
git commit -m "说明这次改了什么"

REM 推送到 GitHub
git push
```

只提交部分文件时，把 `git add .` 换成具体路径，例如：

```bat
git add docs\操作员操作指南.md docs\管理员操作指南.md
git commit -m "docs: 更新操作指南"
git push
```

## 二、Windows：改了前端页面再上传


## 四、Linux：拉取最新代码（最常用）

```bash
# 进入项目目录
cd /home/topuser/productCheck

# 拉取 GitHub 最新代码
git pull
```

**只更新了前端：** 拉完后浏览器按 `Ctrl+F5` 强刷即可，一般不用重启服务。  
**更新了后端：** 见下一节。

---

## 五、Linux：更新后重启后端（改了 Python / 配置时）

```bash
# 进入项目目录
cd /home/topuser/productCheck

# 拉取代码
git pull

# 若 requirements.txt 有变化，再装依赖（没有变化可跳过）
source .venv/bin/activate
pip install -r backend/requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 重启后端（uWSGI）
systemctl restart product-check

# 看是否启动成功
systemctl status product-check

# 测后端接口（本机 5174 端口）
curl http://127.0.0.1:5174/api/health
```

## 七、Linux：Nginx 检查与重载（改了网站配置时）

```bash
# 检查配置有没有写错
/usr/local/nginx/sbin/nginx -t

# 配置正确后再重载（不中断服务）
/usr/local/nginx/sbin/nginx -s reload
```

若你习惯用系统里的 `nginx` 命令，也可以：

```bash
nginx -t
nginx -s reload
```

---

## 八、Linux：看日志排错

```bash
# 后端服务状态
systemctl status product-check

# 后端实时日志（systemd）
journalctl -u product-check -f

# uWSGI 日志
tail -f /home/topuser/productCheck/logs/uwsgi.log

# Nginx 本站点错误日志
tail -f /usr/local/nginx/logs/product_check_error.log
```

按 `Ctrl+C` 可退出实时日志。

## 十一、Linux：防火墙开放前端端口（首次部署时）

```bash

firewall-cmd --permanent --add-port=5173/tcp
```

## 十二、对照表（改了什么 → 跑什么）

| 你改了什么 | Windows | Linux 服务器 |
|------------|---------|--------------|
| 文档 / 说明 | `git add` → `commit` → `push` | `git pull` |
| 前端页面 | 先 `build_frontend.bat`，再提交推送 | `git pull` + 浏览器 `Ctrl+F5` |
| 后端代码 | 提交推送 | `git pull` + `systemctl restart product-check` |
| 依赖包列表 | 提交 `requirements.txt` | `git pull` + `pip install -r ...` + 重启后端 |
| Nginx 配置 | 提交样例文件 | 改服务器实际 conf 后 `nginx -t` 并 reload |

---

*路径或端口若与服务器不一致，以实际为准：前端 5173，后端 5174。*
